## Analyst's final consolidated proposal (verbatim)

### Analysis

**Goal.** Every cargo-dist-built archive AND every native installer (`.deb`, `.rpm`, NSIS `.exe`, `.dmg`) ships with both `yt-dlp` and `deno` at the per-OS path that `crates/app/src/paths.rs::bundled_yt_dlp_path()` / `bundled_deno_path()` expects, so end users (UC 01's non-technical media-savers and podcast rippers) get a no-prerequisites install via their OS's native installer flow.

**Repo state at proposal time (master @ `ffca2c6a3`).**
- `scripts/fetch-deno.sh` exists from UC 05 — argv `<target-triple> <output-dir>`, env `DENO_VERSION`, SHA-only verification, callable in isolation.
- `.github/workflows/release.yml` is dist-managed (regenerated by `dist generate --mode ci`).
- `dist-workspace.toml`: `cargo-dist-version = "0.31.0"`, five targets (`aarch64-apple-darwin`, `aarch64-unknown-linux-gnu`, `x86_64-apple-darwin`, `x86_64-unknown-linux-gnu`, `x86_64-pc-windows-msvc`), `installers = []`, `members = ["cargo:."]`.
- `public.key` (upstream yt-dlp's GPG public key) at repo root.
- `crates/app/src/paths.rs` — per-OS bundled-path resolution with macOS dev-fallback (next-to-binary when `Contents/` parent absent).
- `THREATS.md` § T11 documents the deno SHA-only-vs-yt-dlp-GPG asymmetry.
- `README.UI.md` already contains "Release builds will bundle Deno alongside yt-dlp" (future-tense; UC 06 makes it true).
- No `tests/` dir; no shell-test infra; Justfile has no `test-scripts` recipe.

**cargo-dist 0.31.0 splice mechanism (verified at the v0.31.0 tag).** The release.yml.j2 template iterates `global-artifacts-jobs` and emits `custom-<name>` GHA jobs that `uses: ./.github/workflows/<name>.yml`, gated to run AFTER `build-local-artifacts` + `build-global-artifacts` and BEFORE `host`. Listing our package jobs there keeps them inside the dist-managed dependency graph: `host` waits for them via its `if:` condition, their uploaded artifacts join the same GH release, and `dist generate --mode ci` regenerates `release.yml`'s `needs:` graph automatically. **This is the only post-build splice cargo-dist 0.31.0 exposes; we do not hand-roll a sibling workflow outside the dist graph.**

The cargo-dist 0.31.0 `InstallerStyle` enum is `{Shell, Powershell, Npm, Homebrew, Msi, Pkg}`. None of `.deb` / `.rpm` / `.dmg` / NSIS `.exe`. Path B produces those formats via `global-artifacts-jobs` referenced workflows with our own packagers.

**AC narrowings under path B (signed off by team-lead).**
- **AC #5** keeps its original wording: `<App>.app/Contents/Resources/{yt-dlp,deno}` for `.dmg`; `<install-dir>\{yt-dlp,deno}.exe` for NSIS; `/opt/yt-dlp-ui/{yt-dlp,deno}` for `.deb`/`.rpm` (with `/usr/bin/yt-dlp-ui` shell wrapper).
- **AC #11 PowerShell carve-out** stays in spec — bats SHA-mismatch + GPG-failure tests for `fetch-yt-dlp.sh` are hermetic and unconditional on Linux runners; PowerShell smoke is argv-only.
- **AC #7 `dist build --artifacts=local` smoke** extends to all four installer formats: hermetic check on the appropriate host OS verifies binaries land at the expected paths with the executable bit set.

##### 1.3 Fetch scripts (NEW)

- **`scripts/fetch-yt-dlp.sh`** (Linux/macOS). Argv `<target-triple> <output-dir>`. Env `YT_DLP_VERSION` required (exit 65 if unset). Mirrors `fetch-deno.sh`'s structure (`set -euo pipefail`, `mktemp -d`, EXIT trap).
  - Asset-name map (verify against `gh release view <YT_DLP_VERSION> --repo yt-dlp/yt-dlp --json assets --jq '.assets[].name'` at implementation time):
    - `x86_64-unknown-linux-gnu` → `yt-dlp_linux`
    - `aarch64-unknown-linux-gnu` → `yt-dlp_linux_aarch64`
    - `x86_64-apple-darwin` → `yt-dlp_macos` (universal2)
    - `aarch64-apple-darwin` → `yt-dlp_macos`
    - else → exit 64.
  - Fetches the binary + `SHA2-256SUMS` + `SHA2-256SUMS.sig` from `https://github.com/yt-dlp/yt-dlp/releases/download/<YT_DLP_VERSION>/`.
  - GPG verification: `GNUPGHOME=$(mktemp -d)` (isolated keyring), `gpg --import <REPO_ROOT>/public.key`, `gpg --verify SHA2-256SUMS.sig SHA2-256SUMS`. `REPO_ROOT` env-var override; falls back to deriving from the script's `dirname`.
  - SHA verification (single-line grep + manual compare; `sha256sum -c SHA2-256SUMS` does NOT work — `SHA2-256SUMS` is a multi-entry standard-format file and `-c` would fail on missing peer assets):
    ```bash
    EXPECTED_SHA="$(grep " ${ASSET}\$" "${WORK_DIR}/SHA2-256SUMS" | awk '{print $1}')"
    [[ -z "${EXPECTED_SHA}" ]] && { echo "error: ${ASSET} not in SHA2-256SUMS" >&2; exit 72; }
    ACTUAL_SHA="$(sha256sum "${WORK_DIR}/${ASSET}" 2>/dev/null | awk '{print $1}' || shasum -a 256 "${WORK_DIR}/${ASSET}" | awk '{print $1}')"
    [[ "${EXPECTED_SHA}" != "${ACTUAL_SHA}" ]] && { echo "error: sha256 mismatch for ${ASSET}" >&2; exit 73; }
    ```
  - Moves binary to `<output-dir>/yt-dlp`, `chmod +x`. EXIT trap cleans `GNUPGHOME`.
- **`scripts/fetch-yt-dlp.ps1`** (Windows). Same argv contract. PowerShell idioms (`Set-StrictMode -Version Latest; $ErrorActionPreference = 'Stop'`). Maps `x86_64-pc-windows-msvc` → `yt-dlp.exe`. SHA via `Get-FileHash -Algorithm SHA256` + manual compare. GPG via `gpg.exe` (pinned via `choco install gnupg --no-progress --confirm` in build-setup YAML — no defensive probe). Temp `$env:GNUPGHOME` per-run.
- **`scripts/fetch-deno.ps1`** (Windows). Mirrors UC 05's `fetch-deno.sh` in PowerShell. Maps `x86_64-pc-windows-msvc` → `deno-x86_64-pc-windows-msvc.zip`. SHA-only (deno doesn't sign). `Expand-Archive` to unzip. Moves `deno.exe` to output.

##### 2.1 Three referenced workflows via `global-artifacts-jobs`

Each is `on: workflow_call:` and is registered in `dist-workspace.toml::ci.github.global-artifacts-jobs`. `dist generate --mode ci` regenerates `release.yml` with the right `needs:` references.

###### 2.1.1 `.github/workflows/package-deb-rpm.yml` (NEW)

- `runs-on: ubuntu-latest`. Per-arch matrix on `[amd64, arm64]`.
- SHA-pinned `nfpm` install:
  ```yaml
  env:
    NFPM_VERSION: "2.40.0"
    NFPM_SHA256: "<pinned-at-implementation-time>"
  steps:
    - name: Install nfpm
      run: |
        curl -fsSL "https://github.com/goreleaser/nfpm/releases/download/v${NFPM_VERSION}/nfpm_${NFPM_VERSION}_Linux_x86_64.tar.gz" -o nfpm.tgz
        ACTUAL_SHA="$(sha256sum nfpm.tgz | awk '{print $1}')"
        [[ "${NFPM_SHA256}" == "${ACTUAL_SHA}" ]] || { echo "nfpm SHA mismatch" >&2; exit 73; }
        tar xzf nfpm.tgz nfpm
        sudo mv nfpm /usr/local/bin/nfpm
  ```
- Re-fetches yt-dlp + deno via `bash scripts/fetch-yt-dlp.sh "${target}" runtime-deps/` and `bash scripts/fetch-deno.sh "${target}" runtime-deps/`. Each package job re-fetches because `runtime-deps/` from the pre-build hook does NOT survive into the package job (different runner, fresh checkout).
- Reads cargo-dist's manifest for binary paths (or hardcodes `target/<triple>/release/{app,ad-window}` as fallback). Calls `nfpm package --packager deb --config installer/nfpm.yaml` then `--packager rpm`. Uploads via `actions/upload-artifact`.

###### 2.1.2 `.github/workflows/package-nsis.yml` (NEW)

- `runs-on: windows-latest`. Single arch (`x86_64-pc-windows-msvc`).
- `choco install nsis --no-progress --confirm` (deterministic).
- Downloads Microsoft Evergreen WebView2 Bootstrapper from `https://go.microsoft.com/fwlink/p/?LinkId=2124703` with SHA256 verification (pinned in workflow `env: WEBVIEW2_BOOTSTRAPPER_SHA256`):
  ```yaml
  - name: Download WebView2 Evergreen Bootstrapper
    shell: pwsh
    run: |
      Invoke-WebRequest "https://go.microsoft.com/fwlink/p/?LinkId=2124703" -OutFile MicrosoftEdgeWebview2Setup.exe
      $expected = "${{ env.WEBVIEW2_BOOTSTRAPPER_SHA256 }}"
      $actual = (Get-FileHash MicrosoftEdgeWebview2Setup.exe -Algorithm SHA256).Hash.ToLower()
      if ($expected -ne $actual) { throw "WebView2 bootstrap SHA mismatch" }
  ```
- Re-fetches yt-dlp.exe + deno.exe via the .ps1 scripts.
- Calls `makensis installer/yt-dlp-ui.nsi`. Uploads artifact.

###### 2.1.3 `.github/workflows/package-dmg.yml` (NEW) — universal-binary, single `.dmg`

- `runs-on: macos-latest`. **`needs:` references BOTH macOS triples' `build-local-artifacts` outputs** (aarch64-apple-darwin AND x86_64-apple-darwin). Single job (no matrix split) — produces one universal-binary `.dmg`.
- `actions/download-artifact@v7` with `pattern: artifacts-build-local-{aarch64-apple-darwin,x86_64-apple-darwin}` and `merge-multiple: true` pulls both arches' build outputs.
- Fetches yt-dlp + deno **twice** (once per arch) into per-arch subdirs:
  ```yaml
  - run: bash scripts/fetch-yt-dlp.sh aarch64-apple-darwin runtime-deps/aarch64/
  - run: bash scripts/fetch-deno.sh   aarch64-apple-darwin runtime-deps/aarch64/
  - run: bash scripts/fetch-yt-dlp.sh x86_64-apple-darwin  runtime-deps/x86_64/
  - run: bash scripts/fetch-deno.sh   x86_64-apple-darwin  runtime-deps/x86_64/
  ```
- Calls `installer/build-macos-dmg.sh` which `lipo -create`-merges binaries:
  ```bash
  lipo -create -output "${OUT}/yt-dlp-ui.app/Contents/MacOS/yt-dlp-ui" \
       target/aarch64-apple-darwin/release/app \
       target/x86_64-apple-darwin/release/app
  lipo -create -output "${OUT}/yt-dlp-ui.app/Contents/MacOS/ad-window" \
       target/aarch64-apple-darwin/release/ad-window \
       target/x86_64-apple-darwin/release/ad-window
  lipo -create -output "${OUT}/yt-dlp-ui.app/Contents/Resources/yt-dlp" \
       runtime-deps/aarch64/yt-dlp \
       runtime-deps/x86_64/yt-dlp
  lipo -create -output "${OUT}/yt-dlp-ui.app/Contents/Resources/deno" \
       runtime-deps/aarch64/deno \
       runtime-deps/x86_64/deno
  ```
  Note: `yt-dlp_macos` upstream is already universal2 (single binary works on both arches), so the lipo step on yt-dlp is a no-op-equivalent merge of two identical inputs. `lipo -create` handles this gracefully (output equals input). deno publishes per-arch; lipo produces a real fat binary. Document in build-macos-dmg.sh.
- `hdiutil create -volname yt-dlp-ui -srcfolder <App>.app -ov -format UDZO yt-dlp-ui-universal.dmg`. Single universal-binary `.dmg` output. Uploads artifact.

##### 2.2 nfpm config (NEW `installer/nfpm.yaml`)

Single YAML, two outputs (`.deb` + `.rpm`), per-distro `depends` overrides:

```yaml
name: yt-dlp-ui
version: "${VERSION}"   # set by package-deb-rpm.yml from ${{ github.ref_name }} minus 'v'
arch: "${ARCH}"         # amd64 | arm64, set per matrix split
maintainer: "Harold Hormaechea <hhormaechea@iriusrisk.com>"
description: "Desktop UI wrapping yt-dlp for non-technical users."
license: "PolyForm-Noncommercial-1.0.0"
homepage: "https://github.com/HaroldHormaechea/yt-dlp-ui"
contents:
  - { src: target/${TARGET_TRIPLE}/release/app,         dst: /opt/yt-dlp-ui/yt-dlp-ui,                   file_info: { mode: 0755 } }
  - { src: target/${TARGET_TRIPLE}/release/ad-window,   dst: /opt/yt-dlp-ui/ad-window,                   file_info: { mode: 0755 } }
  - { src: runtime-deps/yt-dlp,                          dst: /opt/yt-dlp-ui/yt-dlp,                      file_info: { mode: 0755 } }
  - { src: runtime-deps/deno,                            dst: /opt/yt-dlp-ui/deno,                        file_info: { mode: 0755 } }
  - { src: installer/launcher.sh,                        dst: /usr/bin/yt-dlp-ui,                         file_info: { mode: 0755 } }
  - { src: LICENSE.UI.md,                                dst: /usr/share/doc/yt-dlp-ui/LICENSE.UI.md }
deb:
  depends: [libwebkit2gtk-4.1-0, libsoup-3.0-0]
overrides:
  rpm:
    depends: [webkit2gtk4.1, libsoup3]
```

Workflow step: `VERSION="${GITHUB_REF_NAME#v}" ARCH=amd64 nfpm package --packager deb --config installer/nfpm.yaml` (per matrix).

**FHS placement.** `/opt/yt-dlp-ui/` is correct per Debian Policy for "third-party self-contained app." `/usr/lib/` is for shared libraries supporting other binaries; `/opt/` is for unbundled self-contained software.

**Per-distro depends.** Debian/Ubuntu names (`libwebkit2gtk-4.1-0`, `libsoup-3.0-0`) for `.deb`; Fedora names (`webkit2gtk4.1`, `libsoup3`) for `.rpm`. Developer verifies actual package names against `dnf search` / `zypper search` output at implementation time.

##### 2.4 NSIS install script (NEW `installer/yt-dlp-ui.nsi`)

- **Per-user install** at `$LOCALAPPDATA\Programs\yt-dlp-ui` — modern convention used by GitHub Desktop, Discord, etc. No UAC prompt; non-technical-user friendly per UC 01 audience.
- **64-bit target** via `Unicode true` + `Target amd64-unicode`.
- Files installed at `$INSTDIR\`:
  - `yt-dlp-ui.exe` (main, from cargo-dist build).
  - `ad-window.exe`.
  - `yt-dlp.exe`, `deno.exe` (from `runtime-deps/`).
  - `LICENSE.UI.md`, `Uninstall.exe` (NSIS auto-generates the uninstaller).
- Start menu shortcut at `Start Menu\Programs\yt-dlp-ui\yt-dlp-ui.lnk`.

##### 2.6 `paths.rs` refactor for testability + `bundled_ad_window_path()` (REQUIRED Rust source change)

The macOS Resources branch must be tested. `current_exe()` is a syscall, not mockable in stable Rust. Refactor:

```rust
fn expected_bundled_path() -> PathBuf {
    let exe_dir = std::env::current_exe()
        .ok()
        .and_then(|p| p.parent().map(std::path::Path::to_path_buf))
        .unwrap_or_else(|| PathBuf::from("."));
    expected_bundled_path_from(&exe_dir, "yt-dlp")
}

fn expected_bundled_path_from(exe_dir: &Path, bin_name: &str) -> PathBuf {
    #[cfg(target_os = "macos")] {
        if let Some(contents) = exe_dir.parent() {
            let resources = contents.join("Resources").join(bin_name);
            if resources.is_file() { return resources; }
        }
        exe_dir.join(bin_name)
    }
    #[cfg(target_os = "windows")] {
        let exe_path = exe_dir.join(format!("{}.exe", bin_name));
        if cfg!(debug_assertions) && !exe_path.is_file() {
            let cmd_path = exe_dir.join(format!("{}.cmd", bin_name));
            if cmd_path.is_file() { return cmd_path; }
        }
        exe_path
    }
    #[cfg(all(not(target_os = "macos"), not(target_os = "windows")))] {
        exe_dir.join(bin_name)
    }
}
```

### Risks & Considerations

1. **Path-A blockers carry forward.** Preflight `dist plan` viability, `include` semantics decision smoke, chmod preservation. All deterministic via local macOS smoke; first-action gate.
2. **`global-artifacts-jobs` is the only post-build splice cargo-dist 0.31.0 exposes.** Verified against the v0.31.0 release.yml.j2 template: the `custom-<name>` job needs `plan` + `build-local-artifacts`, runs in parallel with `build-global-artifacts`, gates `host` via its `if:` condition. No hand-rolled sibling workflow.
3. **`dist generate --mode ci` regenerates `release.yml`.** Mitigated: all hooks live in `ci/dist-build-setup.yml` + the three `package-*.yml` referenced workflows. First post-edit `dist generate` is expected to diff (the splice landing); subsequent runs are no-ops.
4. **`include` is a single global list — no per-target paths.** Mitigated by per-runner staging at `runtime-deps/`. Decision smoke 1 verifies cargo-dist's missing-include behavior. Fallback: canonical-name approach + bounded `paths.rs` Windows-branch edit.
5. **Three orthogonal packagers = three failure modes.** A failed `package-dmg.yml` shouldn't block `.deb` upload. Three independent referenced workflows handle this; release-readiness checklist documented in `scripts/README.md`.
6. **No `.app` bundle from cargo-dist** — we synthesize one. The `installer/build-macos-dmg.sh` script + `Info.plist` are load-bearing for macOS users. Smoke 5 catches drift; documented inline.
7. **`runtime-deps/` re-fetch in each package job.** Each `package-*.yml` runs the fetch scripts again. Wasted bandwidth bounded; rate-limit risk noted but not blocking. yt-dlp `_macos` is universal2 so the lipo merge of two identical inputs is benign.
8. **`Info.plist` version drift.** `CFBundleVersion` templated from git tag via sed in `build-macos-dmg.sh`. Without templating, macOS launchpad shows stale versions.
9. **Apple notarization (Posture 1).** Skipped per Posture 3. `.dmg + .app` workflow is structurally compatible with notarization (just add `xcrun notarytool submit` + stapling steps later); no rework needed when Posture 1 triggers fire.
10. **Linux launcher wrapper at `/usr/bin/yt-dlp-ui`.** Confirmed via `man proc` that `current_exe()` returns the real binary path post-exec, so `paths.rs` Linux branch resolves correctly. Wrapper does not interfere with `current_exe()`.
11. **NSIS WebView2 bootstrap.** Bundled inside the .exe; auto-installs on first launch then deletes from disk. Without it, Win10 users without Edge get a dud install.
12. **Forward-compat for ad-window.** `bundled_ad_window_path()` added now (per challenger Major #8) — future `app → ad-window` spawn code doesn't need to re-litigate UC 06's installer layouts. macOS branch resolves to `Contents/MacOS/ad-window` (helper convention), not `Contents/Resources/`.
13. **CI runtime cost roughly doubles per release.** Path A: 5 jobs. Path B: 5 + 3 packager jobs. macOS minutes (~10× Linux rate) dominate. Bounded to release-tag cadence, not PRs. Acknowledged by team-lead.
14. **yt-dlp release-asset name drift.** Asset-name map verified against `gh release view <YT_DLP_VERSION> --repo yt-dlp/yt-dlp --json assets --jq '.assets[].name'` at the pinned version. Future bumps re-verify.
